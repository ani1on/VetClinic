# План: Миграция с SQLite на PostgreSQL (Supabase)

## Цель
Заменить SQLite на бесплатный PostgreSQL от Supabase, с fallback на SQLite для локальной разработки.

## Изменения

### 1. `back/back_vet/database/core.py` — единая точка подключения к БД
Заменить хардкод SQLite на `DATABASE_URL` из env-переменной с fallback на SQLite:

```python
import os
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from .models import Base

DATABASE_URL = os.environ.get("DATABASE_URL", "sqlite:///./vetclinic.db")

connect_args = {"check_same_thread": False} if "sqlite" in DATABASE_URL else {}

engine = create_engine(DATABASE_URL, echo=False, connect_args=connect_args)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

def init_db():
    Base.metadata.create_all(bind=engine)
```

### 2. `back/back_vet/api/core.py` — убрать дублирующий engine
- Удалить `engine = create_engine("sqlite:///./vetclinic.db", ...)` (строка 34)
- Удалить `Base.metadata.create_all(bind=engine)` (строка 35)
- Импортировать `init_db` из `database.core` вместо локального определения

### 3. `back/main.py` — убрать дублирующий startup
Удалить второй `@app.on_event("startup")` (строки 22-25). `init_db()` уже вызывается в `create_admin_if_not_exists`.

### 4. Настройка Supabase
1. Зарегистрироваться на https://supabase.com
2. Создать новый проект
3. В **Settings** → **Database** скопировать **Connection string** (URI)
4. Формат: `postgresql://postgres.[project-ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres`
5. Добавить в проект на Render env-переменную `DATABASE_URL` со значением из шага 4

### 5. Данные
Таблицы создаются автоматически при первом запуске. Данные из SQLite не мигрируются — потребуется повторная регистрация.

## Файлы
- `back/back_vet/database/core.py` — основное изменение
- `back/back_vet/api/core.py` — удаление дублирующего engine
- `back/main.py` — удаление дублирующего startup

## Верификация
1. Без `DATABASE_URL` → работает SQLite локально
2. С `DATABASE_URL=postgresql+psycopg://...` → работает PostgreSQL
3. На Render: задать env `DATABASE_URL` → таблицы создаются, авторизация работает
