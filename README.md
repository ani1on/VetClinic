# 🐷 FastPig — Ветеринарная цифровая платформа

> **FastPig** — современное веб-приложение для ветеринарной клиники, объединяющее запись на приём, каталог товаров и услуг, личный кабинет с карточками питомцев, избранное, отзывы и мощную панель администратора.

---

## 🌐 Ссылки на production (рабочие версии)

- **Frontend:** [https://fastpig-vetclinic.vercel.app](https://clinicfastpig.netlify.app/)
- **Backend API:** https://api.fastpig-vetclinic.com
- **Swagger документация API:** `https://api.fastpig-vetclinic.com/docs` *(доступна после запуска бэкенда)*

> Если вы разворачиваете проект локально, используйте адреса `http://localhost:8080` (frontend) и `http://localhost:8000` (backend).

## 🚀 Основные функции (ключевой функционал)

### Для клиентов (пользователей)
- **🔐 Авторизация и регистрация** — вход по email/телефону и паролю с отображением силы пароля.
- **📰 Главная лента** — новости клиники, популярные услуги, динамические показатели (количество врачей, средняя оценка, статистика за сегодня).
- **📦 Каталог товаров и услуг** — поиск, фильтр по категориям/наличию, постраничная загрузка, добавление в корзину и избранное.
- **🐾 Личный кабинет** — просмотр и редактирование профиля, управление карточками питомцев (добавление, редактирование, удаление), просмотр корзины и оформление заказов, оставление отзывов с оценкой.
- **📅 Запись на приём** — выбор питомца, услуги, врача, даты (только будние дни, не ранее сегодня) и времени; возможность отменить запись.
- **❤️ Избранное** — сохранение товаров, услуг и врачей в одном списке с просмотром деталей.
- **📋 Отзывы** — отправка отзывов на модерацию, просмотр своих отзывов и удаление их.

### Для администратора
- **📊 Дашборд** — метрики: количество записей, пользователей, товаров, отзывов на модерации и т.д.
- **👥 Управление пользователями** — просмотр, изменение роли (client/admin), удаление.
- **🩺 Управление врачами** — создание, редактирование, архивация врачей (привязка к существующему пользователю).
- **💼 Управление услугами** — добавление, редактирование, архивация услуг (категория, цена, длительность).
- **📦 Управление товарами** — добавление, редактирование, архивация товаров (количество на складе, цена, категория).
- **📰 Управление новостями** — создание, редактирование, публикация/черновик, удаление.
- **⭐ Модерация отзывов** — просмотр всех отзывов, одобрение/отклонение.
- **📅 Просмотр всех записей** — список всех визитов клиентов, изменение статуса записи (ожидает, подтверждено, завершено, оплачено, отменено).
- **🏥 Информация о клинике** — редактирование текста «О нас», контактов, часов работы, соцсетей, лицензий.

## 🛠️ Технологический стек

| Компонент      | Технологии                                                                 |
|----------------|----------------------------------------------------------------------------|
| **Frontend**   | Vue 3 (Composition API), Vue Router, Axios, SCSS/CSS (адаптивный дизайн)   |
| **Backend**    | FastAPI (Python 3.10+), SQLite, Pydantic, python-jose (JWT)                |
| **База данных**| SQLite                                                                     |
| **Аутентификация** | JWT (access + refresh токены), bcrypt для хеширования паролей           |
| **Деплой**     | Frontend – Vercel / Netlify / GitHub Pages; Backend – Render / Heroku / собственный сервер |
| **Доп. инструменты** | VueCLI (front), Uvicorn (back), Git                                  |

## 📡 Основные эндпоинты API

### Аутентификация (`/api/auth`)
| Метод | Эндпоинт               | Описание                            |
|-------|------------------------|-------------------------------------|
| POST  | `/register`            | Регистрация нового пользователя     |
| POST  | `/login`               | Вход (выдаёт access + refresh токены)|
| POST  | `/refresh`             | Обновление access токена            |
| POST  | `/logout`              | Выход (отзыв refresh токена)        |
| POST  | `/forgot-password`     | Запрос на восстановление пароля     |
| POST  | `/reset-password`      | Сброс пароля (по токену)            |
| GET   | `/me`                  | Получить данные текущего пользователя|

|-------|------------------------|-------------------------------------|

## 🖥️ Локальный запуск проекта

### Предварительные требования

- **Node.js** 18+ и **npm** 
- **Python** 3.10+ и **pip**
- **SQLite** 
- **Git**

---

### 1. Клонирование репозитория

```bash
git clone https://github.com/ani1on/VetClinic.git
cd VetClinic
```

### 2. Настройка и запуск бэкенда (FastAPI)

```bash
cd back_vet
# Создать виртуальное окружение
python -m venv venv
# Активировать (Windows)
venv\Scripts\activate
# или (Linux/Mac)
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

Создайте файл `.env` в папке `back_vet` со следующим содержимым:

```env
DATABASE_URL=postgresql://user:password@localhost/dbname
# или для SQLite: sqlite:///./test.db
SECRET_KEY=ваш_секретный_ключ
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=30
```

Выполните миграции (если используются Alembic) или создайте таблицы вручную. Затем запустите сервер:

```bash
uvicorn main:app --reload --port 8000
```

Бэкенд будет доступен по адресу `http://localhost:8000`. Swagger документация: `http://localhost:8000/docs`.

### 3. Настройка и запуск фронтенда (Vue 3)

```bash
cd front
# Установить зависимости
npm install
```

Создайте файл `.env` в папке `front`:

```env
VITE_API_BASE_URL=http://localhost:8000/api
```

Запустите dev-сервер:

```bash
npm run serve
# или
npm run dev
```

Фронтенд будет доступен по адресу `http://localhost:8080`.

---

### 4. Проверка работоспособности

- Откройте `http://localhost:8080` в браузере.
- Зарегистрируйте нового пользователя или войдите с тестовыми данными.
- Для входа в админ-панель измените роль пользователя на `admin` через прямую правку базы данных или создав первого администратора вручную.
